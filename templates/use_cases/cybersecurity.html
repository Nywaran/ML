Use Case 3 — Cybersecurity

How Machine Learning is defending systems against modern cyber threats.

Problem Description

Machine Learning helps cybersecurity systems identify unusual network traffic patterns that may indicate attacks such as DDoS or malware. Unlike traditional methods that rely on known signatures, ML models can detect new and unknown threats by learning from the behavior of the data.

Type of Machine Learning

Unsupervised Learning — models learn what normal behavior looks like and flag anything that deviates significantly from it.

Input Data

Network traffic logs, IP addresses, request frequency, packet sizes, connection times, and user activity patterns.

Expected Output

An alert indicating whether the detected activity is normal or potentially malicious, along with the type of threat identified.

Real-World Relevance

Cyberattacks are growing in frequency and complexity. ML systems can monitor infrastructure 24/7 and respond to threats faster than any human security team.

Real-World Examples

Darktrace uses unsupervised ML to detect cyber threats in real time across thousands of organizations worldwide. CrowdStrike's Falcon platform uses AI to stop breaches by identifying malicious behavior patterns. Google uses ML to filter over 100 million spam and phishing emails every day in Gmail.

Advantages

Detects unknown threats, monitors systems 24/7 without fatigue, responds faster than human analysts, adapts to new attack patterns over time.

Disadvantages

High rate of false positives can overwhelm security teams, black-box models make it hard to understand why a threat was flagged, requires significant computational resources.

Challenges and Limitations

Attackers can use adversarial techniques to fool ML models, the cybersecurity landscape changes rapidly, and there is a shortage of labeled attack data for training.
